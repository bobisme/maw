## SG4 handoff (consumer contract)

**Consumer:** SG4 (`bn-2j45`) / T4.1 reads `ranked_clusters` and picks
hardening targets.

**Top-K rule:** the SG4 loop picks the top-K ranked clusters (K chosen by
SG4 based on hardening capacity). Each picked cluster maps to a
`recommended_fix_class` hint that points at the KIND of hardening
historically known to reduce that cluster (verb-discoverability,
merge-engine-resilience, destroy-guidance-output, etc.). The hint is a
direction, not a prescription — T4.x owns the actual design.

**Validation loop:** post-hardening, T2 re-runs against the same
condition spectrum (C0..C4 × T0..T5) with the hardened maw build. The
loop confirms `total_cost_turns` reduction in the targeted clusters and
does NOT introduce new clusters with higher cost (a hardening pass that
moves friction from one cluster to another is a no-op for SG4 — the
total cost must drop).

**Stop condition:** SG4 declares the hardening pass complete when (a)
the top-3 clusters' aggregate `total_cost_turns` is reduced by ≥ 50%
(SG4-tunable threshold) AND (b) no NEW cluster appears in the top-3
post-hardening.

**Closing this task:** `bn-u9iy` (T2.8) closing **auto-closes SG2
(`bn-2jwi`)** which **unblocks SG4 (`bn-2j45`)**. The auto-close
mechanism is bones' standard child→parent rollup.

## Regenerating this file

```bash

just sg2-friction-list-pilot

just sg2-friction-list <artifact-dir>

cargo run -p maw-bench-metrics --features bench --bin sg2-friction-list -- \
  <artifact-dir> \
  --out-json /path/to/friction-list.json \
  --out-md   /path/to/friction-list.md
```
