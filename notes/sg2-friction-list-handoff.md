# SG2 → SG4 friction-list handoff  (T2.8 / bn-u9iy → T4.1 / bn-2j45)

This document is the **machine-readable + human-readable contract**
between SG2 (agent-ergonomics benchmark) and SG4 (maw hardening). It
pins what SG4/T4.1 reads, what it does with it, and the validation loop
SG4 owns after hardening.

---

## What SG4 reads

**Primary artifact:** the JSON `FrictionList` produced by
`sg2-friction-list` (see `crates/maw-bench-metrics/src/friction_list.rs`,
schema v1). Generated by either:

- `just sg2-friction-list <artifact-dir>` — over real-LLM campaign
  BenchRun JSONs.
- `just sg2-friction-list-pilot` — synthetic-demo bundles
  (TEMPLATE-only; for pipeline validation, not target selection).

**Companion artifact:** `notes/sg2-friction-list.md` — the
human-readable peer (regenerated by the same bin). SG4 reads this for
context; the JSON is the source of truth for automated consumption.

**Schema pin:** `FRICTION_LIST_SCHEMA_VERSION = 1`. Any field rename or
removal MUST bump the version. Additive fields do not.

---

## Consumer contract (what SG4/T4.1 does)

1. **Read the FrictionList JSON.** Assert `schema_version == 1`.
2. **Pick top-K targets.** K is SG4-tunable based on hardening capacity
   (default starting point: K=3 per pass). Each picked target is a
   `FrictionCluster.attribution` value paired with its
   `recommended_fix_class` hint.
3. **Verify the source is acceptable.** If
   `source == "first_pass_classifier"`, SG4 may use it for hardening
   target selection but MUST NOT cite the cost numbers in publication
   text. The publication path requires `source == "blind_double_coded"`
   per pre-reg §6.3.
4. **Pin the harness commit.** Record `harness_commit_sha` in the SG4
   hardening campaign log so the pre/post comparison is rooted at a
   known producer version.
5. **Surface the unattributed bucket.** Even if it's not a hardening
   target itself, SG4 must report `total_unattributed_wasted_turns`
   alongside the targets so the residual blind spot stays visible.

---

## Validation loop (post-hardening)

After SG4 ships a hardening pass:

1. **Re-run T2.6 sweep** with the hardened maw build (same condition
   spectrum: C0..C4 × T0..T5; same per-cell N).
2. **Re-generate the FrictionList** from the new artifact directory.
3. **Diff old vs new.** Tooling: `sg2-friction-list-diff` (future T4.x
   addition; not part of T2.8 scope). Until that ships, manual
   inspection of the two JSONs is the consumer pattern.
4. **Stop conditions** for the hardening pass to be declared complete:
   - **(a) Cost reduction:** top-3 clusters' aggregate
     `total_cost_turns` reduced by ≥ 50% (SG4-tunable).
   - **(b) No regression:** no NEW cluster appears in the top-3
     post-hardening. (A hardening pass that just moves friction from
     one cluster to another is a no-op for SG4.)
   - **(c) Unattributed bucket non-growth:**
     `total_unattributed_wasted_turns` did not grow by more than 20%
     (signals that the heuristic didn't lose ground on classifier
     coverage).

If any stop-condition fails, the hardening pass is incomplete; SG4
iterates.

---

## Hard rules (binding)

- **No composite score.** The FrictionList ranks within the diagnostic
  axis only (per-verb attribution counts, same unit). Cross-axis
  composition (efficiency × correctness × diagnostic) is forbidden by
  the pre-reg §1.2 + §4 and asserted by
  `crates/maw-bench-metrics/tests/no_composite.rs` plus the
  `renderer_emits_no_composite_score` test in `friction_list.rs`. SG4
  must not introduce one when consuming.
- **TEMPLATE marker preserved.** If SG4 quotes pilot numbers in any
  artifact, the TEMPLATE banner MUST be carried over. Publication-grade
  numbers come from a blind-double-coded re-run, NOT the first-pass
  classifier.
- **Top-K is SG4's call, not T2.8's.** T2.8 produces a complete ranked
  list; SG4 chooses how deep to go.

---

## Closing the dependency chain

Closing `bn-u9iy` (this task) auto-closes SG2 (`bn-2jwi`) — the rollup
runs because every child bone of SG2 will then be in `done`:

```
SG2 (bn-2jwi)
├── T2.1..T2.7 (done)
└── T2.8 (bn-u9iy) ← closes here
```

`bn-2jwi → bn-2j45` is a `depends_on` edge; SG2 closing unblocks SG4
(`bn-2j45`). SG4's T4.1 can then begin work on the top-K hardening
targets selected from this friction list.
